# Contributing to Awesome AI Compliance

Thank you for helping maintain this list.

## Guidelines

- Add one resource per pull request when possible.
- Check that the resource is not already in the list.
- Add a short, factual description — no marketing language.
- Place the entry in the most relevant section alphabetically.
- Use this format: `- [Name](URL) - Description.`
- The resource must be publicly accessible.
- Prefer actively maintained projects over abandoned ones.

## Pull Request Process

1. Fork the repository.
2. Create a branch: `Git checkout -b add/resource-name`
3. Make your changes to README.md.
4. Open a pull request with a brief description of what you added and why.

## Questions

Open an issue if unsure where a resource belongs.
